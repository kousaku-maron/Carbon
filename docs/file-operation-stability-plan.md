# ファイル操作安定化計画（サイドバー移動・同期・表示）

## 1. 背景

サイドバーでファイル/フォルダを移動した際に、以下の不安定挙動が発生する。

- 移動元にもファイルが残る（実質的に複製される）
- ノート内容が消えたように見える（実ファイルには内容がある）
- プレビューが空のままになることがある

本計画は、`app` 側のローカルFS同期とエディタ状態遷移を安定化することを目的とする。

## 2. 対象範囲

### 2.1 対象

- サイドバー DnD 移動/リネーム後の保存整合性
- ノート読込・再読込の判定ロジック
- 画像 `asset resolve` 失敗時の表示フォールバック

### 2.2 非対象

- Cloudflare R2/署名URL仕様そのものの変更
- UIデザイン変更（見た目のみの改修）

## 3. 原因整理

### 3.1 旧パスへの遅延保存競合

- `rename` 後、`NoteEditor` の pending save flush が旧パスへ書き戻すケースがある
- 結果として移動元ファイルが再作成され、移動元/移動先の二重状態が起きる

関連箇所:
- `app/src/routes/WorkspaceRoute.tsx`
- `app/src/components/NoteEditor.tsx`
- `app/src/lib/notePersistence.ts`

### 3.2 再読込スキップ条件の誤判定

- `note.id` と `editor.getHTML()` だけで再読込可否を判断しているため、実際は再読込すべきケースでスキップされる
- 表示が空のまま残る要因になる

関連箇所:
- `app/src/components/NoteEditor.tsx`

### 3.3 resolve 失敗時のフォールバック不足

- `carbon://asset/*` 解決失敗時に `setContent` フォールバックがなく、表示が更新されない

関連箇所:
- `app/src/components/NoteEditor.tsx`
- `app/src/lib/assetApi.ts`

## 4. 修正方針

### 4.1 ノート同一性をパスから分離する

- `NoteContent` に `docKey`（セッション中の安定ID）を導入する
- ファイル移動/リネーム時は `path` は更新しつつ `docKey` は維持する
- 別ノート選択時のみ `docKey` を更新する

狙い:
- 「同一ノートのパス変更」と「別ノート切替」を明確に区別する

### 4.2 flush 保存先を文脈で切り替える

- `NoteEditor` は `notePathRef` で現在保存先のみを保持する
- move/rename 時は `notePathRef` を更新し、親から flush を明示実行する

狙い:
- 移動直後の旧パス再作成を防止する

### 4.3 ノート切替時にエディタを再生成する

- `key={docKey}` で `NoteEditor` を再生成する
- 初期本文は `useEditor({ content: markdownToHtml(note.body) })` で直接ロードする

狙い:
- 初期表示/切替表示の分岐を削減し、白画面レースを避ける

### 4.4 resolve 失敗時の明示フォールバック

- 画像 `resolve` は本文全体の再セットではなく、画像ノード属性のみ更新する
- resolve 失敗時は無視して本文表示を維持する

狙い:
- 通信失敗時でも本文表示を維持する

1. 単一ファイル移動
- ルート -> サブフォルダ、サブフォルダ -> ルートの双方で確認
- 移動後に編集・自動保存しても移動元にファイルが再生成されない

2. フォルダ移動
- 開いているノートを含むフォルダを移動
- タイトル/内容/保存先パスが追従する

3. リネーム
- 開いているノートをリネーム
- 直後の自動保存で旧ファイル名が復活しない

4. resolve 異常系
- ネットワーク失敗相当で `resolve` を失敗させる
- 画像は欠けても本文は表示される

5. 連続操作
- 連続で移動 -> リネーム -> 編集を実施
- 内容消失・空表示・二重ファイル化が起きない

## 10. 追加対応（カーソル残留/表示遅延）

### 10.1 症状

- 別ノートを開いた直後、前ノートの内容/カーソルが一時的に残ることがある

### 10.2 原因

- `asset resolve` 完了まで `setContent` を待つ経路があり、描画切替が遅延していた
- ノート切替時にエディタフォーカスが残り、カーソルが残留して見えるケースがあった

### 10.3 対応

- ノート切替時は `NoteEditor` を `key={docKey}` で再生成する
- `useEditor` 初期 `content` に `markdownToHtml(note.body)` を与え、起動直後/切替直後の空表示レースを回避する
- move/rename 追従は `notePathRef` の更新だけに限定し、本文ロードと分離する
- 画像の `resolve` は「本文全体再セット」ではなく、既存画像ノード属性（`src`, `data-asset-uri`）のみ更新する
- 保存は直列キューで統一し、ノート切替時の flush は親コンポーネントから明示実行する

- 再現手順
- `markdownA` を開き本文をクリックしてカーソル表示
- `markdownB` を開く
- 結果: カーソル残留は再現せず、問題解消を確認
