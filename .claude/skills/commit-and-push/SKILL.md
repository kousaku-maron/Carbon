# Commit & Push

変更内容を確認し、シンプルなコミットメッセージを作成してコミット＆プッシュするスキル。

## 実行手順

### 1. 変更内容の確認

以下を並列で実行する:

- `git status` で変更ファイルを確認
- `git diff` と `git diff --staged` で差分を確認
- `git log --oneline -5` で直近のコミットメッセージのスタイルを確認

### 2. コミットメッセージの作成

以下のルールに従ってコミットメッセージを作成する:

- **Eglish**で書く（このリポジトリの慣例）
- 1行で簡潔にまとめる
- 変更の「何を」ではなく「なぜ・何のために」を書く
- 絵文字やprefixは使わない

### 3. ステージング

- `git add` で変更ファイルをステージングする
- `.env`、credentials、秘密鍵など機密ファイルが含まれていないか確認する
- 機密ファイルが含まれている場合は停止してユーザーに警告する

### 4. コミットとプッシュ

- ステージングした差分ファイル一覧とコミットメッセージをユーザーに提示し、確認を得る
- 承認を得たら `git commit` を実行
- 現在のブランチにpushする: `git push origin {current_branch}`
- プッシュが失敗した場合は、force pushせず停止してユーザーに報告する
